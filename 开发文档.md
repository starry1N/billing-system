## 1. 项目概述

计费管理系统是一个面向网吧/机房场景的上机计费管理程序，提供以下核心功能：

| 菜单项 | 功能        |
|--------|-------------|
| 1      | 添加卡      |
| 2      | 查询卡      |
| 3      | 上机        |
| 4      | 下机（自动结算费用）|
| 5      | 充值        |
| 6      | 退费        |
| 7      | 查询统计（消费记录 / 总营业额）|
| 8      | 注销卡      |
| 9      | 计费标准管理|
| 0      | 退出        |

所有数据以**文本文件**持久化到 `data/` 目录，无需数据库。

## 3. 整体架构

系统采用**三层架构**，各层职责严格分离：

```
┌─────────────────────────────────────────────────────┐
│              表示层（Presentation）                  │
│  src/menu/menu.c  ·  include/menu.h                 │
│  负责：用户输入读取、结果展示、菜单调度              │
└───────────────────────┬─────────────────────────────┘
                        │  调用 service 层函数
┌───────────────────────▼─────────────────────────────┐
│              业务逻辑层（Service）                   │
│  card_service.c    billing_service.c                │
│  record_service.c  rate_service.c                   │
│  负责：业务规则校验、状态变更、链表管理              │
└───────────────────────┬─────────────────────────────┘
                        │  调用 dao 层函数
┌───────────────────────▼─────────────────────────────┐
│              数据访问层（DAO）                       │
│  card_file.c    billing_file.c                      │
│  record_file.c  rate_file.c    tool.c               │
│  负责：文本文件读写、序列化/反序列化                 │
└─────────────────────────────────────────────────────┘
                        │  读写
┌───────────────────────▼─────────────────────────────┐
│                   数据文件层                         │
│  data/card.txt  data/billing.txt                    │
│  data/money.txt  data/rate.txt                      │
└─────────────────────────────────────────────────────┘
```

**跨层调用规则**：表示层只调用 service 层，service 层只调用 dao 层，**不允许跨层调用**（menu.c 不能直接调用 `*_file.c` 中的函数）。

## 4. 数据结构

### 4.1 Card — 卡信息（card.h）

```c
typedef struct {
    char   cardNo[19];      /* 卡号，最多 18 位         */
    char   password[9];     /* 密码，最多 8 位          */
    double money;           /* 当前余额（元）           */
    int    state;           /* 卡状态（见下方常量）     */
    time_t openTime;        /* 开卡时间                 */
    time_t endTime;         /* 截止时间（暂未启用）     */
    time_t lastUseTime;     /* 最近一次上机时间         */
    int    useCount;        /* 累计上机次数             */
    double totalMoney;      /* 累计消费金额（元）       */
    int    delFlag;         /* 删除标识：0正常 / 1已删  */
} Card;
```

卡状态常量：

| 常量                   | 值 | 含义   |
|------------------------|----|--------|
| `CARD_STATE_NORMAL`    | 0  | 未上机 |
| `CARD_STATE_USING`     | 1  | 上机中 |
| `CARD_STATE_CANCELLED` | 2  | 已注销 |

链表结构：`CardNode`（单向链表节点）/ `CardList`（含 `head` 指针和 `count` 计数）

---

### 4.2 Billing — 消费记录（billing.h）

```c
typedef struct {
    char   cardNo[19];   /* 卡号                      */
    time_t startTime;    /* 上机时间                  */
    time_t endTime;      /* 下机时间（上机中时为 0）  */
    double amount;       /* 消费金额（元）            */
    int    state;        /* 0=未结算 / 1=已结算       */
    int    delFlag;      /* 删除标识                  */
} Billing;
```

---

### 4.3 Money — 充值/退费记录（money.h）

```c
typedef struct {
    char   cardNo[19];   /* 卡号       */
    double amount;       /* 金额（元） */
    int    type;         /* 1=充值 / 2=退费 */
    time_t opTime;       /* 操作时间   */
    int    delFlag;      /* 删除标识   */
} Money;
```

---

### 4.4 Rate — 计费标准（rate.h）

```c
typedef struct {
    double ratePerHour;  /* 每小时费率（元/小时），默认 1.5 */
    time_t updateTime;   /* 最后更新时间                    */
} Rate;
```

## 5. 模块详解

### 5.1 表示层 — menu

**文件**：`src/menu/menu.c`，`include/menu.h`

每个菜单函数对应一个功能，负责：
1. 打印提示并读取用户输入
2. 调用对应 service 层函数
3. 根据返回值打印成功/失败信息

| 函数                       | 对应菜单 | 调用的 Service 函数      |
|----------------------------|----------|--------------------------|
| `outputMenu()`             | 主菜单   | —                        |
| `addCardMenu()`            | 1. 添加卡 | `addCard()`             |
| `queryCardMenu()`          | 2. 查询卡 | `getCard()`, `queryCards()` |
| `loginMenu()`              | 3. 上机   | `loginCard()`           |
| `logoutMenu()`             | 4. 下机   | `logoutCard()`          |
| `rechargeMenu()`           | 5. 充值   | `rechargeCard()`        |
| `refundMenu()`             | 6. 退费   | `refundCard()`          |
| `queryStatisticsMenu()`    | 7. 查询统计 | `queryBillings()`, `getTotalRevenue()` |
| `cancelCardMenu()`         | 8. 注销卡 | `cancelCardService()`   |
| `rateManagementMenu()`     | 9. 计费标准 | `getRateValue()`, `setRate()`, `printCurrentRate()` |

#### 输入验证与异常处理

表示层的所有涉及卡号输入的函数都实现了**三重防护**：

**1. 长度限制（缓冲区溢出防护）**
```c
scanf("%18s", cardNo);  /* 最多读取 18 个字符，防止溢出 cardNo[19] */
```

**2. 输入异常检查 + 非空检查**
```c
if (scanf("%18s", cardNo) != 1) {
    clearInputBuf();    /* 清空输入缓冲（防止后续输入错乱） */
    printf("输入无效！\n");
    return;             /* 立即返回，不执行后续操作 */
}
if (strlen(cardNo) == 0) {
    printf("卡号为空！\n");
    return;
}
```

**3. 超长输入检测（防止残留输入串联）**
```c
int ch = getchar();
if (ch != '\n' && ch != EOF) {
    clearInputBuf();
    printf("卡号长度不能超过18位！\n");
    return;
}
```

**防护覆盖的菜单函数**：
- ✅ `addCardMenu()` — 卡号/密码支持超长检测、非空与异常检查
- ✅ `queryCardMenu()` — 卡号关键字支持超长检测，超长时立即提示异常并返回
- ✅ `loginMenu()` — 卡号、密码检查
- ✅ `logoutMenu()` — 卡号、密码检查
- ✅ `rechargeMenu()` — 卡号、密码、金额检查
- ✅ `refundMenu()` — 卡号、密码、金额检查
- ✅ `cancelCardMenu()` — 卡号、密码检查

**返回值约定**：
| 场景 | 含义 | 处理方式 |
|---------|------|---------|
| `scanf(...) != 1` | 输入失败（如输入非数字给 `%d`） | 调用 `clearInputBuf()`，提示错误，返回 |
| `strlen(...) == 0` | 空字符串（防御性检查） | 提示为空并返回 |
| 读取后存在残留字符 | 用户输入超过字段长度上限 | 调用 `clearInputBuf()`，提示“长度不能超过X位”，返回 |

### 5.2 业务逻辑层 — service

#### card_service.c

| 函数签名 | 说明 |
|----------|------|
| `void initCardList(CardList *list)` | 初始化空链表 |
| `void freeCardList(CardList *list)` | 释放链表所有节点内存 |
| `void getCard(CardList *list)` | 从文件读取全部有效卡到链表 |
| `int addCard(Card *card)` | 添加卡（检查唯一性，填充系统字段后写文件）|
| `Card *queryCard(CardList *list, const char *cardNo)` | 精确查找，返回节点数据指针 |
| `void queryCards(CardList *list, const char *keyword, CardList *result)` | 模糊查找，结果写入 result 链表（调用方 `freeCardList`）|

`addCard()` 返回值：`1` 成功 / `-1` 卡号已存在 / `0` 文件写入失败

---

#### billing_service.c

| 函数签名 | 说明 |
|----------|------|
| `int loginCard(const char *cardNo, const char *password, Card *outCard)` | 上机：验证、改状态、创建未结算消费记录 |
| `int logoutCard(const char *cardNo, const char *password, double *cost, time_t *outEndTime)` | 下机：计算费用、扣款、结算消费记录 |
| `Billing *queryBillings(int *count)` | 读取所有有效消费记录（堆内存，调用方 `free`）|
| `double getTotalRevenue(void)` | 汇总所有已结算消费记录的金额 |

`loginCard()` 返回值：`1` 成功 / `0` 卡号或密码错误 / `-2` 卡状态不对 / `-3` 余额不足  
`logoutCard()` 返回值：`1` 成功 / `0` 卡号或密码错误 / `-2` 卡未处于上机状态

**扣费公式**：

$$费用 = \frac{(下机时间 - 上机时间)}{3600} \times 当前费率$$

若计算费用 > 余额，则只扣光余额（不透支）。

---

#### record_service.c

| 函数签名 | 说明 |
|----------|------|
| `int rechargeCard(const char *cardNo, const char *password, double amount, Card *outCard)` | 充值并记录 |
| `int refundCard(const char *cardNo, const char *password, double amount, Card *outCard)` | 退费并记录 |
| `int cancelCardService(const char *cardNo, const char *password, double *refundAmount)` | 注销卡（退还余额后标记为注销状态）|

`rechargeCard()` 返回值：`1` 成功 / `0` 卡号或密码错误 / `-2` 卡已注销  
`refundCard()` 返回值：`1` 成功 / `0` 卡号或密码错误 / `-2` 卡不在正常状态 / `-3` 退费金额超过余额  
`cancelCardService()` 返回值：`1` 成功 / `0` 卡号或密码错误 / `-2` 卡正在使用

| 函数签名 | 说明 |

## 6. 数据文件格式

所有文件均为**纯文本**，每行一条记录，字段以 `|` 分隔，时间字段存储 Unix 时间戳（`time_t` 转 `long`）。

### data/card.txt

```
cardNo|password|money|state|openTime|endTime|lastUseTime|useCount|totalMoney|delFlag
```

示例：
```
test|123|0.00|2|1773043885|0|0|0|0.00|0
```

---

### data/billing.txt

```
cardNo|startTime|endTime|amount|state|delFlag
```

- `state`：0=未结算，1=已结算
- `endTime`：上机中时为 0

---

### data/money.txt

```
cardNo|amount|type|opTime|delFlag
```

- `type`：1=充值，2=退费

---

### data/rate.txt

单行：
```
ratePerHour|updateTime
```

示例：
```
2.0000|1773045090
```

## 7. 函数返回值约定

| 返回值 | 含义（service 层通用语义）|
|--------|--------------------------|
| `1`    | 操作成功                 |
| `0`    | 卡号/密码错误，或文件操作失败 |
| `-1`   | 数据重复（如卡号已存在） |
| `-2`   | 业务状态不满足（卡已注销/卡正在上机等）|
| `-3`   | 余额不足或金额超限       |

DAO 层函数一般只返回 `1`（成功）或 `0`（失败）。

## 8. 关键业务流程

### 8.1 上机流程

```
loginMenu()
  └─ loginCard(cardNo, password, &outCard)
       ├─ 读取所有卡 getCard(&list)
       ├─ 校验卡号+密码
       ├─ 校验状态 == CARD_STATE_NORMAL
       ├─ 校验余额 > 0
       ├─ 更新卡：state=USING, lastUseTime=now, useCount++
       ├─ updateCard(card)
       └─ saveBilling()  // 创建 state=未结算 的消费记录
```

### 8.2 下机流程

```
logoutMenu()
  └─ logoutCard(cardNo, password, &cost, &endTime)
       ├─ 读取所有卡 getCard(&list)
       ├─ 校验卡号+密码
       ├─ 校验状态 == CARD_STATE_USING
       ├─ 计算 fee = (now - lastUseTime) / 3600 * getRateValue()
       ├─ fee = min(fee, card->money)
       ├─ 更新卡：money -= fee, totalMoney += fee, state=NORMAL
       ├─ updateCard(card)
       └─ updateBillingByCard()  // 将未结算记录更新为已结算
```

### 8.3 注销卡流程

```
cancelCardMenu()
  └─ cancelCardService(cardNo, password, &refundAmount)
       ├─ 校验卡号+密码
       ├─ 校验状态 != CARD_STATE_USING (上机中不可注销)
       ├─ refundAmount = card->money
       ├─ 若余额 > 0，写 money.txt 一条退费记录
       ├─ card->money = 0, card->state = CANCELLED
       └─ updateCard(card)
```

## 9. 编译与运行

### 方式一：双击 build.bat

直接运行 `build.bat`，编译成功后在 `计费管理系统/` 目录下生成 `AMS.exe`，双击或在命令行运行即可。

```bat
build.bat
```

### 方式二：手动 g++ 命令

```bat
cd 计费管理系统
g++ src/main.c src/dao/tool.c src/dao/card_file.c src/dao/billing_file.c ^
    src/dao/record_file.c src/dao/rate_file.c ^
    src/service/card_service.c src/service/billing_service.c ^
    src/service/record_service.c src/service/rate_service.c ^
    src/menu/menu.c ^
    -o AMS.exe -finput-charset=UTF-8 -fexec-charset=GBK -Wall -w
```

> **字符集说明**：`-finput-charset=UTF-8` 表示源文件为 UTF-8；`-fexec-charset=GBK` 表示运行时使用 GBK，适配 Windows 控制台（代码页 936）。若更换终端，需同步修改此参数。

### 运行要求

- 以 `计费管理系统/` 作为**工作目录**运行，否则 `data/` 目录路径无法找到。
- 首次运行会自动创建 `data/` 目录。
- GCC 路径在 `build.bat` 中的 `set GCC=` 行定义，可按实际安装路径修改。

## 10. 已知限制与后续开发建议

### 卡号限制功能（已实现）

系统已对卡号输入实现了完整的防护机制：

| 防护措施 | 实现位置 | 说明 |
|--------|--------|------|
| **长度上限** | `menu.c` 全部菜单 | 通过 `scanf("%18s")` 限制卡号为最多 18 位 |
| **超长输入拦截** | `addCardMenu()`、`queryCardMenu()` | 读取后检查换行；若存在残留字符则提示长度超限并返回 |
| **缓冲区溢出** | `card.h` 常量定义 | `CARD_NO_LEN=19` （18位卡号 + `\0`） |
| **非空检查** | `addCardMenu()` 等 | 通过 `strlen(cardNo) == 0` 检查空串 |
| **输入异常检查** | 所有菜单函数 | 通过检查 `scanf()` 返回值捕获输入失败 |
| **缓冲区恢复** | 工具函数 `clearInputBuf()` | 输入异常时清空缓冲，防止后续菜单混乱 |
| **唯一性检查** | `card_service.c: addCard()` | 防止创建重复卡号 |

**防护覆盖范围**：卡号、密码、金额等所有用户输入字段

### 当前其他限制

| 限制 | 说明 |
|------|------|
| 文件读取上限 | `card_file.c` 中 `lines[1024][256]`，最多支持 1024 条卡记录；`billing_file.c` 中 `lines[4096][128]`，最多 4096 条消费记录 |
| 并发不安全 | 文件操作无锁，多进程同时运行可能导致数据损坏 |
| 密码明文存储 | 密码直接以明文写入 `card.txt`，存在安全风险 |
| Windows 专用 API | `_mkdir` 是 Windows 专属，跨平台移植需适配 |
| `endTime` 字段未使用 | `Card.endTime` 暂未实现到期逻辑 |
| 卡号格式无限制 | 允许卡号包含任意字符（建议添加格式检查） |

### 后续开发建议

1. **扩展记录上限**：将 `card_file.c` 的静态数组改为动态分配（`malloc` + `realloc`），或迁移到 SQLite 等嵌入式数据库。
2. **密码加密**：使用 MD5 或 SHA-256 对密码做哈希，避免明文存储。
3. **日志功能**：新增 `log_file.c`，记录每次操作的时间、操作类型和操作人，便于审计。
4. **到期管理**：启用 `Card.endTime` 字段，在上机时检查卡是否过期。
5. **多用户权限**：引入管理员角色，区分普通用户和管理员权限（如只有管理员可修改计费标准）。
6. **报表导出**：在"查询统计"中增加导出为 CSV 功能。
7. **跨平台移植**：将 `_mkdir` 替换为条件编译（`#ifdef _WIN32` / `#else mkdir`）。
8. **单元测试**：为各 service 层函数编写测试用例，构建简单的自动化回归测试。
9. **卡号格式验证**：可选在 `card_service.c` 中添加卡号格式检查（如仅允许数字），增强数据合规性。
